Required safeUrl helper (copy the exact function from src/components/speakers/SocialLinks.astro:20-31 — DO NOT skip; this is the T-05-02 mitigation):
```ts
function safeUrl(raw: string | undefined): string | null {
  if (!raw) return null;
  const trimmed = raw.trim();
  if (!trimmed) return null;
  try {
    const u = new URL(trimmed);
    if (u.protocol === "http:" || u.protocol === "https:") return u.toString();
    return null;
  } catch { return null; }
}
```

Required safeLogoPath helper (NEW — T-05-03 mitigation for sponsor logos):
```ts
function safeLogoPath(raw: string): string | null {
  const t = (raw || "").trim();
  if (!t) return null;
  // Accept absolute public paths ("/sponsors/foo.svg")
  if (t.startsWith("/") && !t.startsWith("//")) return t;
  // Accept https absolute URLs
  try {
    const u = new URL(t);
    if (u.protocol === "https:") return u.toString();
  } catch {}
  return null;
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CSV `url` column → SponsorCard anchor href | External/admin data becomes click destination |
| CSV `logo` column → `<img src>` | External/admin data becomes image source |
| CSV `description_fr/en` → DOM text + title attribute | External text surfaces to the user |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02 | Injection / XSS | SponsorCard anchor href | mitigate | `safeUrl(sponsor.data.url)` allowlist (http/https only) called before any href is rendered; null return disables the link rather than failing the page. Pattern copied verbatim from SocialLinks.astro (Phase 11). |
| T-05-03 | Injection | SponsorCard img src | mitigate | New `safeLogoPath()` helper accepts only `/`-prefixed absolute public paths or `https://` URLs. Rejects `javascript:`, `data:`, `//protocol-relative`, `../` traversal. Null return renders the sponsor name as text instead of a broken image. |
| T-05-06 | XSS | description_fr/en → title attribute | accept | Astro auto-escapes `{expression}` bindings on attributes. The `title` attribute is not executed by browsers (unlike href); it's safe surface for adversarial text beyond visual disruption. No `set:html` used anywhere in this plan. |
| T-05-07 | Repudiation | CTA mailto link | accept | Plain mailto: with hardcoded address; no user-supplied content in the link. Spoofing concern is the user's mail client, not our site. |
| T-05-08 | Information disclosure | rel="noopener" on external anchor | mitigate | `rel="noopener noreferrer"` on every sponsor anchor — prevents reverse-tabnabbing and referrer leakage (matches Phase 12 security posture for ticketing link). |
</threat_model>
