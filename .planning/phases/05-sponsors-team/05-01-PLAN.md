<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Google Sheets CSV → build pipeline | Admin-edited data crosses into site at build time via fetchCsvOrFallback. CSV values flow into Zod, then into Astro components. |
| CSV `url` column → rendered anchor href (downstream, in Plan 05-02) | Admin-supplied URL string becomes DOM attribute. Must pass safeUrl() allowlist. |
| CSV `logo` column → `<img src>` attribute (downstream, in Plan 05-02) | Admin-supplied path becomes image source. Must validate no schemes other than `/`-prefixed public paths or `https://`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | Google Sheets CSV source | accept | Build-time only fetch; no runtime attack surface. SPONSORS_CSV_URL / TEAM_CSV_URL env vars documented to only accept Google Sheets publish-to-web URLs. Compromise of the sheet = repo-level trust issue (same as any admin content); Zod schema rejects malformed rows at build time, failing the deploy. |
| T-05-02 | Injection | CSV `url` column → anchor href | mitigate (deferred to Plan 05-02) | Plan 05-02 SponsorCard MUST route sponsor.url through safeUrl() before rendering, rejecting javascript:/data:/vbscript: schemes. This plan annotates the schema with a security note. |
| T-05-03 | Injection | CSV `logo` column → img src | mitigate (deferred to Plan 05-02) | Plan 05-02 SponsorCard MUST validate that logo is either (a) a `/`-prefixed absolute path into /public or (b) an https:// URL — reject all other schemes. This plan flags the threat; schema left as permissive string to keep the data layer generic. |
| T-05-04 | Injection | CSV social_* columns → anchor href | mitigate | Zod `socialUrl` parses as `z.string().url()` (rejects malformed strings at build). Downstream SocialLinks.astro applies safeUrl() allowlist (http(s) only). Double-layered defense already in place — no new work. |
| T-05-05 | XSS | CSV text fields (name, description, role) → DOM text | accept | Astro auto-escapes all `{expression}` interpolations. Risk: only if a future dev uses `set:html` directive — call it out in SponsorCard/TeamMemberCard review (covered in Plans 05-02/05-03 threat models). |
</threat_model>
