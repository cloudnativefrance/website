Create `docs/updating-content.md`. This is the largest file in the doc set — per D-03 it must be readable by non-dev content editors (2-3 people) AND developers. Plain language, no jargon unless defined. NO screenshots (D-03 — they rot against Sheets UI churn). Cite `src/content.config.ts` as the Zod source of truth rather than duplicating schemas verbatim (D-05).

Structure:

1. **H1: Updating Content** — one-paragraph intro: "All speaker, session, sponsor, and team data lives in Google Sheets. The site fetches each sheet as CSV at build time; if the remote is unreachable the build falls back to a committed copy under `src/content/`. This page walks through how to edit each dataset and trigger a rebuild."

2. **H2: The four datasets** — a table with 4 rows:

   | Dataset | Google Sheet (publish-to-web CSV) | Env var that overrides it | Local fallback path |
   |---|---|---|---|
   | Speakers | linked from sheet tab — ask a maintainer for the URL | `SCHEDULE_SPEAKERS_CSV_URL` | `src/content/schedule/speakers.csv` |
   | Sessions | linked from sheet tab — ask a maintainer for the URL | `SCHEDULE_SESSIONS_CSV_URL` | `src/content/schedule/sessions.csv` |
   | Sponsors | not yet wired to a remote sheet — edit the local fallback directly | `SPONSORS_CSV_URL` (unset by default) | `src/content/sponsors/sponsors.csv` |
   | Team | not yet wired to a remote sheet — edit the local fallback directly | `TEAM_CSV_URL` (unset by default) | `src/content/team/team.csv` |

   One-line explainer below the table: "Env-var precedence lives in `src/lib/remote-csv.ts`. Default URLs for speakers and sessions currently point to the CND 2026 sheet; sponsors and team default to empty, meaning the build reads the committed fallback directly."

3. **H2: How the build resolves content** — short bulleted explanation:
   - If `<DATASET>_CSV_URL` is set and fetch succeeds → remote CSV wins.
   - If unset OR fetch fails (network error, 404, timeout after 8 seconds, malformed response) → local fallback CSV under `src/content/` wins.
   - Build logs prefix each decision with `[csv]` and say whether remote or fallback was used.
   - Results are memoized per process, so one build sees one consistent snapshot across all pages.

4. **H2: Speakers** — per-entity section. This is the main worked example; be thorough.
   - **Where it lives**: `src/content/schedule/speakers.csv` (fallback) + the speakers tab of the CND Google Sheet (remote).
   - **Columns** — list them with a one-line semantic and required/optional flag. Source of truth: the Zod schema in `src/content.config.ts`. Use this table (mirrors `speakers` collection):
     - `slug` — required, URL-safe identifier; becomes `/speakers/{slug}`. Use kebab-case (e.g. `vincent-behar`).
     - `name` — required, human-readable name shown in UI.
     - `photo_url` — optional, absolute URL or site-relative path to a headshot.
     - `company` — optional.
     - `role` — optional, e.g. "DevRel / Docker Captain".
     - `bio` — optional, 2-3 sentence biography.
     - `twitter`, `linkedin`, `github`, `bluesky`, `website` — optional URLs; must be valid URLs or empty. Invalid URLs fail the build.
     - `keynote` — optional boolean (`TRUE` / `FALSE` / `1` / `0` / empty). Marks a speaker as a keynote speaker.
   - **Add a new speaker — step by step** (the literal runbook):
     1. Open the CND Google Sheet; go to the Speakers tab.
     2. Append a new row at the bottom.
     3. Fill in `slug` (required, kebab-case, unique) and `name` (required). Leave optional columns empty if you don't have the value yet — the build tolerates empty cells but rejects invalid URLs.
     4. For social links, paste the full URL (e.g. `https://linkedin.com/in/jane`, not `@jane`).
     5. Save. Sheets auto-saves so nothing more to do in the UI.
     6. If the Sheet has never been published: File → Share → Publish to web → choose the Speakers sheet → CSV → Publish. Copy the URL that Sheets gives you. Ask a maintainer to set `SCHEDULE_SPEAKERS_CSV_URL` to this value in the deployment environment. If the sheet is already published, nothing to do here — the publish URL is stable.
     7. Trigger a rebuild (see §How to trigger a rebuild in production below).
   - **Validation note**: the Zod schema in `src/content.config.ts` fails the build if a required column is missing or a URL is malformed. Check CI logs if the build fails — the error message will name the offending row.

5. **H2: Sessions** — mirror the speakers structure:
   - Where it lives: `src/content/schedule/sessions.csv` / Sessions tab in the Sheet.
   - Columns (authoritative: header of `src/content/schedule/sessions.csv`): `id` (required, URL-safe), `title` (required), `speakers` (required, comma-separated list of speaker slugs — each must exist in the speakers dataset), `track`, `level`, `room`, `format` (e.g. `keynote`, `talk`, `lightning`), `start_time` (required, ISO 8601 with timezone, e.g. `2026-02-03T09:00:00+01:00`), `duration_min`, `tags` (comma-separated), `feedback_url`, `slides_url`, `recording_url` (Phase 8 uses this to switch the session into "watch replay" mode), `cover_image_url`, `language` (`fr` or `en`), `status` (e.g. `confirmed`, `pending`), `description` (markdown-ish, supports multi-line when CSV-quoted).
   - Add a new session — step by step:
     1. Ensure all speaker slugs you'll reference already exist in the Speakers tab. Cross-references are resolved at build time; a missing slug breaks the schedule.
     2. Append a row with at minimum `id`, `title`, `speakers`, `start_time`.
     3. If the talk has a co-speaker, list both slugs comma-separated in one cell: `arthur-outhenin-chalandre,quentin-swiech`.
     4. `status` = `confirmed` to publish the session; anything else keeps it in pipeline-only state (check code for current gating — the canonical filter lives in `src/lib/schedule.ts`).
     5. Save. Rebuild.

6. **H2: Sponsors** — per-entity section. Until `SPONSORS_CSV_URL` is wired, edits happen directly in `src/content/sponsors/sponsors.csv` via a PR.
   - Columns (authoritative: `sponsors` collection in `src/content.config.ts`): `id` (required), `name` (required), `tier` (required, one of `platinum` / `gold` / `silver` / `community` — unknown values fail the build), `logo` (required, path or URL), `url` (required, must be a valid URL), `description_fr` (required), `description_en` (required).
   - Add a new sponsor — step by step:
     1. Obtain the logo as SVG, place it under `public/sponsors/{slug}.svg` (or reference an external URL in the `logo` column).
     2. Edit `src/content/sponsors/sponsors.csv`, append one row with all required columns.
     3. Open a PR with the logo asset + the CSV change in the same commit (CSV schema rule from CLAUDE.md).

7. **H2: Team** — per-entity section. Same model as sponsors — no remote sheet yet, edits via PR against `src/content/team/team.csv`.
   - Columns (authoritative: `team` collection in `src/content.config.ts`): `id` (required), `name` (required), `role_fr` (required), `role_en` (required), `group` (required, one of `core` / `program-committee` / `volunteers`), `photo` (optional), `social_linkedin`, `social_github`, `social_bluesky`, `social_twitter`, `social_website` (all optional URLs).
   - Add a new team member — step by step:
     1. Add a headshot to `public/team/{id}.jpg` if you have one (optional).
     2. Append a row to `src/content/team/team.csv` with required columns filled.
     3. Open a PR.

8. **H2: How to trigger a rebuild in production** — short section: "The site is deployed via GitOps from the separate `cnd-platform` repo. Pushing an updated Docker image to `ghcr.io` is handled automatically by `.github/workflows/build-image.yml` on every merge to `main`. To pick up a Google Sheet change without a code change, trigger a manual run of the workflow from the GitHub Actions tab, or push a no-op commit to `main`. For full Flux CD details and rollout controls, see the `cnd-platform` repo."

9. **H2: Troubleshooting** — short list of common failure modes and fixes:
   - **Build fails on a Zod error** — a required column is empty, a URL column has a malformed value, or `tier`/`group` has an unknown enum value. Check the error message for the row id; fix the Sheet.
   - **New entry not appearing on the site** — confirm the rebuild ran (`.github/workflows/build-image.yml`) and check the build logs for `[csv]` messages indicating remote vs. fallback.
   - **Remote fetch silently using fallback** — check build logs for `[csv] {label}: remote fetch failed (...); using local fallback`. Usually a timed-out publish URL; republish the Sheet.

10. **H2: Further reading** — link back to `src/content.config.ts` (authoritative Zod schemas per D-05), `src/lib/remote-csv.ts` (fetch logic + env vars), `CONTRIBUTING.md`, `docs/testing.md`.
