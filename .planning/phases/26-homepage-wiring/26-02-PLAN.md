<flag_specification>
French national flag (drapeau tricolore) — official specification:

| Band | Position (left -> right) | Hex | Name |
|------|---------------------------|-----|------|
| 1    | Left third                | #002654 | Bleu de France (deep blue) |
| 2    | Middle third              | #FFFFFF | White |
| 3    | Right third               | #ED2939 | Rouge de France (red) |

Aspect ratio of the actual flag is 2:3 (width:height). For favicon use we use a SQUARE
1:1 viewBox so the flag renders cleanly in a tab favicon (which is rendered into a square
slot by every browser). Each band is therefore 1/3 of the square width.

This is the same colour set used on the official Élysée Palace and Government of France
brand assets (lighter saturation than the older 1976-2020 "navy + red" variant).

NO prefers-color-scheme variants — a national flag does not have a dark-mode form.

  <action>
Replace the ENTIRE contents of `public/favicon.svg` with the SVG body below. This is a wholesale file rewrite (the current monogram + prefers-color-scheme media query has nothing to preserve).

Use this exact file content:

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 3 3" shape-rendering="crispEdges">
    <title>Drapeau de la France — Cloud Native Days France</title>
    <rect x="0" y="0" width="1" height="3" fill="#002654" />
    <rect x="1" y="0" width="1" height="3" fill="#FFFFFF" />
    <rect x="2" y="0" width="1" height="3" fill="#ED2939" />
</svg>
```

Notes on the choices:
- `viewBox="0 0 3 3"` — square 1:1 so the flag fills the favicon slot (browsers render favicons into a square box; a 2:3 viewBox would leave letterboxing).
- `shape-rendering="crispEdges"` — keeps the band boundaries pixel-sharp at the small sizes a favicon renders at (16×16, 32×32, 48×48). Without this hint browsers may anti-alias the boundaries to a fuzzy line.
- Each `<rect width="1" height="3">` covers exactly one third of the viewBox width, matching the flag's 1/3 + 1/3 + 1/3 vertical-band layout.
- `<title>` element provides accessibility text and bilingual hint (browsers expose this to AT users who inspect the favicon link).
- Colours are the contemporary French government palette (#002654 / #FFFFFF / #ED2939) — same blue + red used on Élysée brand assets.

DO NOT:
- Add a `prefers-color-scheme` media query. A national flag has no dark-mode variant. The previous monogram needed one because it was a single-colour glyph; a tricolor doesn't.
- Use a 2:3 (real-flag) viewBox. Browsers render favicons into a square box and a 2:3 SVG would either letterbox or stretch.
- Delete or modify `public/favicon.ico`. It stays as the legacy-browser fallback. Regenerating .ico from this SVG requires a graphics pipeline (ImageMagick / Pillow) outside the agent's surface; the user can drop a custom .ico later if pixel-perfect legacy parity matters. The Layout already declares both icons (`<link rel="icon" type="image/svg+xml" href="/favicon.svg" />` followed by `<link rel="icon" type="image/x-icon" href="/favicon.ico" />`), so modern browsers preferring the SVG will pick up the new flag immediately while old IE/legacy clients keep showing the existing .ico.
- Touch `src/layouts/Layout.astro`. The existing `<link>` declarations on lines 92-93 already point at the right path; the swap is purely the file body.
- Add any external `<image href="...">` references — favicon must be self-contained inline SVG (no network fetch on every tab load).
