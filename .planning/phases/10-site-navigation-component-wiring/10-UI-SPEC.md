## Spacing Scale

Tailwind 4 base unit 4px. DESIGN.md-defined scale. All values are multiples of 4.

| Token | Value | Usage in Phase 10 |
|-------|-------|-------------------|
| xs | 4px | Underline offset below active nav link |
| sm | 8px | Icon-to-text gap inside CTA button; mobile menu row inner padding |
| md | 16px | Horizontal padding on nav container (mobile); mobile menu link vertical padding (12px → rounded to 16px for touch) |
| lg | 24px | Gap between desktop nav links (`gap-6`) |
| xl | 32px | Horizontal padding on nav container (desktop, `px-8` equivalent within `max-w-7xl`) |
| 2xl | 48px | Reserved (not used in this phase) |
| 3xl | 64px | Reserved (not used in this phase) |

**Exceptions:**
- Mobile hamburger button touch target: **44px × 44px** (DESIGN.md accessibility rule: minimum 44px touch targets on mobile)
- Header total height: **56–64px** (logo `h-8` to `h-10` plus `py-3` = 56px) — not a spacing token, but a derived fixed dimension

Phase-specific application of the 60/30/10 split already established in DESIGN.md.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-background` oklch(16.8% 0.052 286.4) / #0e0a24 | Page surface behind the header (when scrolled the header blurs over this) |
| Secondary (30%) | `--color-background/90` with `backdrop-blur-sm` | Sticky header surface; mobile menu panel; TranslationNotice banner background uses `--color-card` #1a1240 |

**Accent reserved for (explicit list — Phase 10 only):**
1. Active nav link text color
2. Active nav link bottom border (2px, D-03)
3. Register CTA button background
4. Focus rings on nav links, hamburger button, and CTA (`--color-ring` = primary)

**Not accent-eligible in this phase:** inactive nav links (use `--color-muted-foreground`), hamburger icon (use `--color-foreground`), scroll border (use `--color-border`), TranslationNotice (uses `--color-card` + `--color-foreground`).

**Warm pink (`--color-accent` #ff8a9e) is NOT used in Phase 10.** DESIGN.md reserves it for "one accent per visual section maximum"; the nav already claims its accent slot with primary blue for consistency with interactive conventions.

## Interaction Contract

| Behavior | Trigger | Visual Response | Implementation |
|----------|---------|-----------------|----------------|
| Nav link hover (desktop) | Pointer enter | Text color transitions `--color-muted-foreground` → `--color-foreground` over 150ms | Tailwind `hover:text-foreground transition-colors` |
| Active page indicator | Page matches link `href` | Text becomes `--color-primary`; 2px bottom border `--color-primary` with 4px offset (`pb-0.5`) | `class:list` with `isActive()` helper (exact match for home, `startsWith` for sections) |
| Scroll border | `window.scrollY > 0` | Header gains `border-b` in `--color-border`; fades in over 200ms | Vanilla JS `scroll` listener (passive); toggles `border-border` / `border-transparent` classes |
| Mobile menu open | Tap hamburger button | Panel slides down from header; body gets `overflow-hidden`; hamburger icon swaps to close (X) | Vanilla JS `click` handler; `classList.toggle("hidden")` on panel + body lock |
| Mobile menu link tap | Tap link inside open menu | Menu closes; page navigates | Link `<a>` navigation is default; menu does not need JS to close (page reload clears state) |
| Focus (keyboard) | Tab key lands on link/button | 2px `--color-ring` outline with 2px offset | Tailwind `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2` |

**Mobile menu pattern chosen (D-02 discretion):** Hamburger + slide-down panel (below the header, full-width). Rationale: 6 nav items + CTA fit comfortably stacked; slide-down is less modal-heavy than full-screen overlay and matches existing sticky-header aesthetic. Avoids drawer library dependency.

**TranslationNotice display logic chosen (D-07 discretion):** Blanket non-default-language approach — rendered on every page where `resolvedLang !== defaultLang` (i.e., all `/en/` pages). Simpler than per-page fallback detection; honest given partial translation state.

## Responsive Contract

| Breakpoint | Nav Layout |
|------------|-----------|
| `< 768px` (mobile) | Logo (left) + hamburger + LanguageToggle (right). Desktop links hidden. Tap hamburger → panel slides below header with stacked links + CTA. |
| `>= 768px` (`md`) | Logo (left) + horizontal nav links (center, `gap-6`) + CTA + LanguageToggle (right). Hamburger hidden. |

**Container:** `max-w-7xl mx-auto px-4 md:px-6` inside a full-width sticky `<header>`.

**Logo sizing:**
- Mobile: use `logo-notext.svg`, `h-8` (32px)
- Desktop (`md:`): use `logo.svg` (with text), `h-8` (32px), `md:h-10` (40px)

## Accessibility Contract

| Element | Requirement | Source |
|---------|-------------|--------|
| Touch targets | Minimum 44px × 44px on mobile (hamburger, CTA, nav links in mobile panel) | DESIGN.md §Accessibility |
| Color contrast | All nav text meets WCAG 2.1 AA (muted-foreground 5.5:1, primary 6:1 on background) | DESIGN.md §Accessibility |
| Focus indicators | Visible 2px `--color-ring` with 2px offset on all interactive elements | DESIGN.md §Accessibility |
| Keyboard navigation | Tab order: logo → nav links (desktop) OR hamburger (mobile) → CTA → LanguageToggle | Standard |
| ARIA | Hamburger button: `aria-label="Menu"`, `aria-expanded={state}`, `aria-controls="mobile-menu"` | WAI-ARIA Authoring Practices |
| Semantic HTML | `<header>` wraps, `<nav>` for link list, `<a>` for links (not `<button>`) | HTML5 |
| Color-alone | Active state uses BOTH color (primary) AND shape (bottom border) — never color alone | DESIGN.md §Accessibility |
