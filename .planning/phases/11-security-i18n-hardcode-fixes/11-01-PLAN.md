<!-- Contract that downstream code will call. Extracted from src/i18n/utils.ts. -->

From src/i18n/utils.ts:
```typescript
export function useTranslations(lang: Locale): (key: keyof (typeof ui)[typeof defaultLang]) => string;
// Falls back to ui[defaultLang][key] when ui[lang][key] is undefined.
```

Keys are typed via `keyof (typeof ui)[typeof defaultLang]` — i.e., the FR branch is the source of truth for valid key names. If a key is missing in FR, TypeScript rejects `t("that.key")`. If a key is missing in EN (but present in FR), there is NO compile error — the fallback silently returns the FR value. **Therefore: every new key MUST be added to both fr AND en.**
