<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| i18n dictionary (ui.ts) → HTML attribute serialization | `t("schedule.format.*")` and `t("schedule.agenda.remove")` values flow into Astro `{expr}` attribute context — auto-escaped. |
| HTML attribute → client-side JS via `getAttribute` | Values read back in the inline `<script>` are plain strings (no HTML parsing). |
| Client-side JS → `element.innerHTML` template literal | The `refreshAgenda()` function (line 1124) uses `item.innerHTML = \`...${agendaRemoveLabel}...\`` — this IS an HTML sink. Translation values interpolated here must not contain attacker-controlled HTML. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-06 | Tampering (stored XSS via translation value injected into innerHTML) | `refreshAgenda()` line 1129 template literal | mitigate | **Primary mitigation**: translations are authored in `src/i18n/ui.ts`, committed via PR, code-reviewed — no runtime user input. **Defensive posture**: the new key `schedule.agenda.remove` is a short plain noun ("Retirer" / "Remove") with no HTML metacharacters; values are pinned by code review. **Residual accepted**: if a future maintainer adds HTML to a translation value, this sink would execute it. Acceptable for a static site with a code-reviewed i18n dictionary; not acceptable if translations become user-sourced — flag for revisit if i18n ever gains a CMS upstream. |
| T-11-07 | Tampering (XSS via user-supplied URL in SocialLinks) | SocialLinks.astro href attributes | mitigate (already in place) | `safeUrl()` scheme allowlist (http/https only) — verified intact by Task 2 audit. Closes CR-01. |
| T-11-08 | Information disclosure (format labels reveal nothing sensitive) | format chip labels | accept | Labels are public UI copy; no security-relevant information. |
| T-11-09 | Tampering (format label data-* attribute manipulation at runtime via DOM-XSS) | `.schedule-root` data attributes | accept | Attributes are server-rendered + static; the only path to manipulate them post-render is a DOM-XSS vector elsewhere, which would already be game-over. Not Phase 11's responsibility. |
</threat_model>
