  <name>Task 1: Create design system in Google Stitch with CND France brand colors</name>
  <files>DESIGN.md</files>
  <action>
    Use Google Stitch MCP tools to create the design system. The visual identity must be:

    **Color Palette** (derived from CND France logo):
    Logo colors extracted from src/assets/logos/principal/logo.svg:
    - #cf0822 — Vibrant red (CND signature)
    - #4985e8 — Bright blue (cloud/tech feel)
    - #20134d — Deep purple (depth, premium)
    - #b8deff — Light blue (sky, openness)
    - #ff8a9e — Warm pink (community warmth)
    - #ffffff — White (text on dark)

    Map these into a dark-theme design system:
    - Background: Deep dark derived from #20134d (the logo's deep purple)
    - Primary: #4985e8 blue (main interactive color, CTAs)
    - Accent: #cf0822 red or #ff8a9e pink (warm community accents, highlights)
    - Foreground: Near-white for readability on dark backgrounds
    - Cards/surfaces: Slightly lighter than background, still within the purple-dark family
    - Muted: Desaturated versions for secondary text and borders

    **Typography:**
    - Font: DM Sans (brand continuity requirement)
    - Weights: 400 (body), 500 (emphasis), 600 (subheadings), 700 (headings)
    - Scale: Choose a bold/energetic type scale -- larger headings, generous spacing
    - Suggested sizes: 14px body, 16px large body, 20/24/30/36/48px heading scale

    **Mood:** Bold and energetic -- vibrant colors, strong contrasts, dynamic feel (think KubeCon energy). Dark theme with technical aesthetic softened by warm community accents.

    **Geometric patterns:** Choose a pattern style that complements the palette. Options include hex mesh (cloud-native reference), abstract polygons, gradient lines, or circuit-board patterns. The pattern should work as a subtle background texture, not competing with content.

    **Spacing:** Use a 4px base unit scale (4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80, 96).

    **Border radius:** 6px default (slightly rounded, modern but not bubbly).

    Steps:
    1. Use `create_design_system` MCP tool to define the system with the above parameters
    2. Use `create_project` + `generate_screen_from_text` to create a sample screen showing the design system applied (hero section mockup with heading, subheading, CTA button, card component)
    3. Use `get_screen` to export the design and extract the final token values
    4. Write DESIGN.md to the project root with all token values, rationale, and component patterns

    If Stitch MCP tools are unavailable or produce unsatisfactory results, manually author DESIGN.md based on the logo color analysis above, converting hex values to OKLCH for Tailwind 4 compatibility.

    DESIGN.md must include these sections:
    - Color Palette (with OKLCH values for each semantic color)
    - Typography (font, weights, size scale)
    - Spacing Scale
    - Border Radius
    - Component Patterns (button variants, card styling, badge styling)
    - Geometric Background Pattern description
    - Dark Theme Rationale
