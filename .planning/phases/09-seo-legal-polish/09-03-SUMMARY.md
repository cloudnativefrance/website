# Phase 09 Plan 03: Homepage Event JSON-LD Summary

**One-liner:** Both homepage variants now emit a valid schema.org Event JSON-LD block (generated by a typed helper) and set `og:type="event"`; every other page keeps `og:type="website"` untouched.

## What Shipped

### Task 1 — `src/lib/event-schema.ts` helper (commit `660252c`)

New file exports two functions:

- `buildEventSchema({ lang, siteUrl, description })` → typed object literal with all D-07 fields (name, startDate/endDate 2027-06-03, eventAttendanceMode=Offline, eventStatus=Scheduled, Place with PostalAddress + GeoCoordinates 48.8899/2.3702, Organization, Offer, absolute image URL, inLanguage derived from lang).
- `eventSchemaJson(input)` → thin `JSON.stringify` wrapper for inline injection.

Trailing-slash handling: strips one trailing slash from `siteUrl` before interpolating into `image` and `organizer.url`, so `https://cloudnativedays.fr/` and `https://cloudnativedays.fr` both produce `https://cloudnativedays.fr/og-default.png` (no double slash).

Security note inline in the file documents T-09-08 (no user input reaches serializer; if a future caller passes untrusted text, re-audit for `</script>` escape).

### Task 2 — Homepage wiring (commit `fb76d8b`)

Both `src/pages/index.astro` and `src/pages/en/index.astro` now:

1. Import `eventSchemaJson` from `@/lib/event-schema`.
2. Derive `description = t("seo.default_description")` and `siteUrl = Astro.site.toString()` (with production fallback).
3. Call `eventSchemaJson({ lang, siteUrl, description })`.
4. Pass `description` + `ogType="event"` to `<Layout>` so Layout's meta-description and og:type reflect the homepage.
5. Emit `<script type="application/ld+json" set:html={schemaJson} />` as first child of Layout's slot (inside `<body>`, before `<main>` — JSON-LD position is SEO-agnostic per Google guidance).

`set:html` is required — Astro would otherwise HTML-escape the quotes in `schemaJson`, breaking the JSON parser.

## Threat Surface

Plan threats T-09-08 / T-09-09 / T-09-10:

- **T-09-08 (Tampering / XSS via `set:html`):** mitigated. `schemaJson` is produced exclusively by `JSON.stringify(buildEventSchema(...))` where every field is either a hardcoded constant or a closed-enum i18n string (`seo.default_description`). `JSON.stringify` guarantees proper JSON-escape of `"`, `\`, and control characters; no `</script>` sequences in any literal or i18n value. Source-code comment on `eventSchemaJson` documents the invariant for future callers.
- **T-09-09 / T-09-10:** accepted per plan — ticketing URL and organizer URL are public information / build-time constants.

No new threat surface introduced beyond what the plan's threat model anticipated.

## Known Stubs

- **`offers.price: "0"`** — placeholder. Event is not actually free; once real pricing is known, update the literal in `src/lib/event-schema.ts` or drop the `offers` block entirely until the ticket shop is ready. Not a correctness blocker (schema still validates), but search engines may surface "Free" in rich results based on this value.

- **JSON-LD on other routes** — intentionally homepage-only per D-07. If speakers/sessions ever need per-entity JSON-LD (`Person`, `PresentationDigitalDocument`), the helper pattern established here is the template.
