# Phase 8 — UI Design Contract

> Visual and interaction contract for Phase 8: Event Lifecycle. Covers (1) CFP status section on homepage (EVNT-04), (2) post-event rewire of CountdownTimer to locale-aware `/replays` (EVNT-01), (3) new `/replays` page in FR + EN (EVNT-01). EVNT-02 is already shipped (Phase 7); no new UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>

## Phase Scope — What We're Designing

Three UI surfaces, in priority order:

1. **CFP Status Section (homepage)** — Full-width section slotted between Key Numbers and Footer on `/` and `/en/`. Three visual states driven by two ISO date constants: `coming-soon`, `open`, `closed`.
2. **CountdownTimer post-event CTA (rewire)** — Existing component at `src/components/hero/CountdownTimer.tsx:71-85`. Only the `href` target changes: `#replays` → `/replays` (FR) or `/en/replays` (EN). No visual change.
3. **`/replays` page (FR + EN)** — New static Astro page, mirrors `/venue` + `/en/venue` pattern. Grid of recording cards grouped by track. Hidden (unlinked, excluded from sitemap) until `Date.now() > TARGET_DATE`.

### Accent reserved for
Explicit list (no "all interactive elements"):
- CFP status pill background **when and only when** `state === "open"`
- CFP section primary CTA button ("Submit a Talk" / "Soumettre une conférence") — this is the one accent per-section allowance from DESIGN.md
- Nothing else in this phase. The `/replays` page uses `--color-primary` (blue) for its CTAs, not accent.

### CFP state → color mapping

| State | Pill bg | Pill text | CTA variant | CTA label source |
|-------|---------|-----------|-------------|------------------|
| `coming-soon` | `--color-secondary` (muted dark) | `--color-muted-foreground` | Ghost button, disabled-looking but enabled (link to Conference Hall page if URL set, else hidden) | `cfp.cta.notify` |
| `open` | `--color-accent` (pink) | `--color-accent-foreground` (dark) | Accent button (pink), glow shadow `--shadow-glow-accent` | `cfp.cta.submit` |
| `closed` | `--color-muted` (dark) | `--color-muted-foreground` | Section degrades: either entirely hidden OR muted-only no-CTA note (planner decides per D-04). Both copy variants required in i18n. | `cfp.closed.note` (no CTA) |

Destructive color (CND Red) is intentionally never used in this phase — CFP "closed" is a neutral lifecycle state, not an error.

## Component Inventory

Per-surface component breakdown. All components reuse existing DS primitives — no new base components introduced.

### 1. CFP Status Section (homepage)

Located between Key Numbers block and Footer in `src/pages/index.astro` and `src/pages/en/index.astro`.

**Structure:**
- Section wrapper: full-width, `--spacing-12` top/bottom padding, `--color-background`, optional subtle hex-mesh pattern at 8% opacity consistent with hero
- Inner container: `max-w-[1280px]`, centered, `--spacing-4` horizontal padding on mobile
- Content stack (vertical, center-aligned on mobile, left-aligned from `md` up):
  - CFP status pill (top): small badge, `--radius-full`, `4px 10px` padding, `--text-xs` weight 400 uppercase. Icon (lucide `Megaphone` for coming-soon, `Send` for open, `Archive` for closed) + label.
  - Section headline: `--text-2xl` weight 700, token `cfp.heading`
  - Description paragraph: `--text-base` weight 400, `--color-muted-foreground`, max 2 lines on desktop, token `cfp.description.<state>`
  - CTA button (primary slot): see state → color mapping table. Button height 52px (`lg` size), `--radius-md`, icon + label (lucide `ArrowRight` trailing)
  - Secondary meta row (small, `--text-xs` weight 400 muted): displays deadline when state=`open` (e.g. "Closes April 30, 2027") — computed from `CFP_CLOSES` constant in user's locale via `Intl.DateTimeFormat`

**States:**
- `coming-soon` — pill muted, "Coming soon" label, description promises announcement, CTA ghost or hidden
- `open` — pill accent-pink, "CFP open" label, description invites submission, CTA accent button with glow
- `closed` — either section is fully hidden (`return null`) OR rendered as a single muted line: "CFP closed — thanks to all submitters. See you June 3, 2027." Planner picks which; both i18n keys must exist.

**Responsive:**
- Mobile (<640px): center-aligned stack, CTA full-width, meta row below CTA
- Tablet (640-1024px): left-aligned content, CTA inline-size
- Desktop (≥1024px): two-column optional — text left, decorative element (hex cluster or simple illustration) right at 40% width; planner/Stitch decides

**Motion:** on state change at runtime (e.g. user leaves tab open across midnight), no animation — the page doesn't need live ceremony. On initial paint, pill and CTA may fade in with 200ms ease if section uses existing scroll-reveal pattern elsewhere on homepage; otherwise no animation.

### 2. CountdownTimer post-event CTA (rewire only)

File: `src/components/hero/CountdownTimer.tsx:71-85`.

**Change:** `href="#replays"` → `href={lang === "en" ? "/en/replays" : "/replays"}`.

**No visual change.** Existing classes (`bg-primary`, `shadow-[0_0_20px_...]`, `text-lg font-bold`) are kept byte-for-byte. This is wire-only, not a visual redesign.

### 3. `/replays` page (FR + EN)

Files: `src/pages/replays/index.astro` + `src/pages/en/replays/index.astro` (mirrors `/venue` + `/en/venue` pattern).

**Structure:**
- Page header (top, `--spacing-24` top padding, `--spacing-12` bottom):
  - H1: `--text-4xl` weight 700, token `replays.heading` ("Replays" / "Replays") — same word both locales OR localize per planner
  - Lead paragraph: `--text-base` weight 400 `--color-muted-foreground`, max ~70 chars/line, token `replays.lead`
  - "Back to schedule" ghost link (top-left or below lead): lucide `ArrowLeft` icon + label, token `replays.back_to_schedule`
- Track groups (main content, `--spacing-12` gap between groups):
  - Each group has a sub-heading (`--text-2xl` weight 700) with a 4px-wide left border in the track's `trackColor()` hue, `--spacing-3` padding-left
  - Below sub-heading: responsive grid of recording cards
- Footer spacer: `--spacing-24` bottom padding before site Footer

**Recording card:**
- Container: `--color-card` background, `--radius-lg` (8px), `1px solid --color-border`, `--spacing-6` padding, full-card clickable link to YouTube session URL
- Left accent: 4px solid vertical bar in `trackColor(track)` hue, flush-left inside the card (inset `--spacing-6` from content)
- Top line: track badge pill — `--text-xs` weight 400 uppercase, bg = track color at 15% opacity, text = track color at 100%, `--radius-full`, `4px 10px` padding
- Title: `--text-base` weight 700, `--color-card-foreground`, 2-line clamp with `line-clamp-2`
- Speakers row: `--text-xs` weight 400 `--color-muted-foreground`, 1-line truncate, comma-separated
- Watch indicator (bottom-right of card OR right side of content): lucide `Play` icon in a small circle bg=`--color-primary` at 15% opacity, icon color=`--color-primary`, 32px diameter. Visible label reads "Watch replay" / "Regarder le replay" (see copywriting contract).
- Hover: border transitions to `--color-primary` at 50% opacity (existing DS card hover pattern), 200ms ease. Optional: `translateY(-2px)` with `--shadow-md` lift.
- Focus: DS focus ring (`--color-ring`, 2px offset)
- Minimum touch target: entire card is the link, card height ≥ 120px — well above 44px

**Grid responsive:**
| Breakpoint | Columns | Gap |
|------------|---------|-----|
| <640px | 1 | `--spacing-4` (16px) |
| 640-1024px | 2 | `--spacing-6` (24px) |
| ≥1024px | 3 | `--spacing-8` (32px) |

**Embeds:** cards link out to `youtube.com/watch?v=…` (user-initiated click = opt-in per D-10). If any inline preview is added later, must use `youtube-nocookie.com` — no exceptions.

**States:**
- Has recordings: render grouped grid (above)
- Empty (build-time: zero sessions have `recording_url`): render single centered block — muted heading + body, token `replays.empty.heading` / `replays.empty.body`, optional "Back to home" ghost link. Page still builds but only reachable via direct URL.
- Pre-event (runtime: `Date.now() < TARGET_DATE`): page exists but is unlinked from nav, excluded from sitemap via `@astrojs/sitemap` filter. If reached directly, render normally (empty state likely) — no special "coming soon" takeover.

**Navigation integration:** `src/components/Navigation.astro` gains a conditional `/replays` entry. Rendered via tiny client-side enhancement (reveal after `Date.now() > TARGET_DATE`) OR build-time flag — planner decides. Link label: `nav.replays` i18n key (new).
