## Architectural decisions locked by this plan

### AD-01 — New component, not a variant prop
Create **`src/components/past-editions/PastEditionMinimal.astro`** rather than extend `PastEditionSection.astro` with a `variant="minimal"` flag.

**Why:** The shell consumed by 2026 + venue has 7 optional slots (rail, h2, stats, photos, video, brandCallout, galleryCta). A "minimal" variant would need to suppress 5 of them AND impose a 2-column structural layout that the shell doesn't support. Adding a `variant` prop plus layout branching would bloat the shell with ~40 lines of conditionals that only benefit this single 2023-homepage use case. A dedicated ~35-line component is lighter, easier to test, and leaves the shell focused on the full/2026 case.

### AD-02 — Photo selection: 01, 05, 08
From the existing 6-photo homepage subset (01, 03, 05, 07, 08, 10), keep `kcd2023_01`, `kcd2023_05`, `kcd2023_08`:
- **01** — "full room wide shot" (scale signal)
- **05** — "attendee discussions" (human/hallway-track moment)
- **08** — "technical workshop" (content depth)

Drops 03/07/10 (thematically redundant: 03 is another hallway-track, 07 is another wide shot, 10 is another atmosphere shot).

### AD-03 — Logo asset location
Logo is already imported in `src/lib/editions-data.ts` as `kcdLogo` from `@/assets/logos/kcd2023/logo-color.png` and exposed as `EDITION_2023.brandLogo`. No new import needed — the minimal component reuses this field.

### AD-04 — Responsive behavior
2-column layout at `md:` and up via `grid md:grid-cols-2 gap-8 md:gap-12`. Below `md`, single column stacks LEFT (photos + logo) above RIGHT (link). LEFT column uses a 3-column photo grid (`grid-cols-3 gap-3`) with the logo below, centered.

### AD-05 — i18n keys
Reuse existing `editions.2023.video_cta` ("Voir la playlist YouTube 2023" / "Watch the 2023 YouTube playlist") as the visible link text. No new copy needed. The existing `editions.2023.thumbnail_alt.{1,5,8}` alts are reused. Logo alt reuses the hardcoded string `"Kubernetes Community Days France 2023"` already used in the full-shell brandCallout.

### AD-06 — Anchor preservation
The `<section>` rendered by `PastEditionMinimal.astro` accepts an `id` prop and renders `<section id={id}>`, identical to the shell's D-08 Phase 17 extension. Homepage mount passes `id="edition-2023"`.
</architectural_decisions>
