## Overview

The v1.0 milestone delivers a complete bilingual (French/English) marketing and programme site for Cloud Native Days France 2027, held June 3, 2027 at CENTQUATRE-PARIS. Built on Astro 6 + React islands + Tailwind CSS 4 + shadcn/ui, designed in Google Stitch, deployed as a Docker/Nginx image on Kubernetes via the sibling `cnd-platform` GitOps repo.

The product surface covers:

- Hero landing with event identity, live countdown, registration CTA, and key numbers
- Speaker directory and individual profiles with talk cross-references
- Tiered sponsor showcase (Platinum / Gold / Silver / Community) and team directory
- Venue page with map, transport, accommodation, and accessibility info plus previous-edition recap
- Interactive schedule with track filtering, parallel-track timeline, bookmarkable personal agenda (localStorage), iCal export, and Open Feedback deep links
- Event lifecycle adaptation: pre-event countdown flips to post-event "watch replays" mode, CFP status indicator tied to Conference Hall, per-talk YouTube recording links
- SEO/legal polish: Open Graph, Twitter cards, JSON-LD Event schema, hreflang, sitemap, robots.txt, Code of Conduct, Privacy, Terms, footer
- Navigation component, security/i18n hardening, brand completion, schema-drift cleanup, and full contributor documentation (README, repo-structure, CSV update runbook, testing, CONTRIBUTING)

Key architectural spine: CSV-sourced-of-truth content pipeline (Google Sheets → CSV → Zod-validated loaders → Astro content collections) with local fallback files, Stitch-first visual workflow with DESIGN.md handoff, single-locale schema with fallback, and a runtime TARGET_DATE pattern for event-lifecycle rendering.

## Milestone Summary

**Key Decisions:**

- **CSV source of truth**: Content for speakers, sessions, sponsors, team is authored in Google Sheets, published as CSV, pulled at build via env vars (`SCHEDULE_SESSIONS_CSV_URL` etc.), with local fallback files committed to the repo. Non-technical organizers can edit content without a PR.
- **Stitch-first visual workflow**: Every new page or significant UI change is designed in Google Stitch first, validated by the user, then implemented in code. DESIGN.md is the handoff artifact. Enforced via CLAUDE.md.
- **Single-locale schema with fallback**: Content collection entries exist per locale (`fr/` and `en/` subfolders) with fallback to FR when an EN entry is absent.
- **Runtime TARGET_DATE pattern**: Event lifecycle (pre-event countdown vs post-event replays) is driven by a runtime date constant, not build-time flags, so the same artifact transitions automatically on June 3, 2027.
- **GitOps deployment split**: K8s manifests live in a separate `cnd-platform` repo; this repo produces only the Docker image (9.5MB alpine:3.21 + nginx). FNDN-07 (K8s manifests) is therefore deferred out of this milestone by design.
- **Tailwind 4 CSS-native @theme + OKLCH**: Tokens live entirely in `global.css` via CSS-native `@theme`; no JS config file. Colors use OKLCH for perceptual uniformity and wide-gamut readiness.
- **Astro content collections + Zod**: Invalid content fails the build, not production. Flat schemas preferred over nested to match CSV column shape.

**Notable Phases:**

- **Phase 7 (Interactive Schedule)**: Highest-complexity feature, deliberately isolated. Delivers filtering, parallel-track timeline, localStorage bookmarks, iCal export, mobile view, and Open Feedback deep links in a single plan.
- **Phase 8 (Event Lifecycle)**: Introduced the runtime TARGET_DATE pattern and went through a dedicated Stitch-first UI-SPEC gate (6 screens, 6/6 dimensions passed) before implementation — the template for future design-heavy phases.
- **Phases 13 + 14 (gap closure)**: Added after the v1.0 audit surfaced pre-existing TypeScript/test drift and missing contributor documentation. Unblocked the milestone's "passed" audit verdict.

**Issues Resolved:**

- Speaker schema drift (33 astro-check errors + 2 broken test suites) closed in Phase 13
- Integration gaps (orphan components, missing navigation) closed in Phase 10
- Security/i18n hardcodes (SocialLinks XSS, hardcoded 'fr'/'en' strings) closed in Phase 11
- Missing project documentation surface closed in Phase 14

**Issues Deferred (v1.1 carry-over):**

- **FNDN-07**: K8s Deployment/Service/Ingress manifests — owned by the `cnd-platform` GitOps repo by design
- **P1 — Subjective visual UATs**: 7 phases still in `human_needed` UAT status (01, 02, 03, 04, 05, 09, 11), ~18 subjective visual tests. Only Phase 08 UAT is complete (5/5).
- **P2 — SPKR-01 fixture drift**: `tests/build/speakers-grid.test.ts` has fixture drift (non-blocker)
- **P2 — Zod 13 LoaderConstraint errors**: `src/content.config.ts` emits type errors that do not affect runtime
- **P2 — Organizer content placeholders**: `CONFERENCE_HALL_URL`, ticketing URL, legal TODOs, social URLs, `og-default.png` all await final organizer input
